# Integrated Multi-Method Pattern Mining for E-commerce Analytics

Course project applying clustering, pattern mining, time series analysis, anomaly detection, and RFM-based business scoring to the Online Retail II dataset.

## Quick Start

### 1. Clone the repo

```bash
git clone <repo-url>
cd <repo-folder>
```

### 2. Set up the virtual environment

**macOS / Linux:**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

**Windows (PowerShell):**
```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

### 3. Install dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 4. Register the Jupyter kernel

```bash
python -m ipykernel install --user --name=ecommerce-mining --display-name "Python (ecommerce-mining)"
```

When you open a notebook, select **Python (ecommerce-mining)** as the kernel.

### 5. Data

Data is already preprocessed and put in data/

---

## Algorithms Used

- **Clustering:** k-means on log-transformed RFM features (`scikit-learn`)
- **Pattern mining:** Segment-conditioned FP-Growth (`mlxtend`)
- **Time series:** Quarterly temporal stability scoring (custom)
- **Anomaly detection:** Isolation Forest on per-invoice features (`scikit-learn`)
- **RFM business decision:** CLV-weighted composite segment scoring (custom)

## Troubleshooting

**`ModuleNotFoundError: No module named 'openpyxl'`**
You skipped step 3. Activate the venv and re-run `pip install -r requirements.txt`.

**Jupyter doesn't show the right kernel**
Re-run step 4. Make sure you've activated the venv first.

**Preprocessing produces way fewer rows than expected**
Check that both Excel sheets (`Year 2009-2010` and `Year 2010-2011`) loaded. If only one loaded, the sheet name probably changed — open the file in Excel to confirm.

**FP-Growth runs forever or eats all your RAM**
The min_support is too low. Start at 0.02 and increase. Smaller segments need lower thresholds, but never go below 0.005 on a full segment.
